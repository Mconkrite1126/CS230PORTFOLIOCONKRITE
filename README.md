# CS230PORTFOLIOCONKRITE

CS230 Portfolio
Briefly summarize The Gaming Room client and their software requirements. Who was the client? What type of software did they want you to design?

The client for this project was The Gaming Room. They wanted to expand their game, Draw It or Lose It, from an Android-only application into a web-based game that could be used on different operating systems and devices. The goal was to create a system that could support multiple players and teams while making sure names stayed unique and the application could continue to grow over time.

What did you do particularly well in developing this documentation?

I think I did a good job comparing the different operating platforms and explaining why one option would work better than another. As the project went on, I became more comfortable discussing topics like operating systems, memory management, storage, security, and distributed systems. I also feel like I got better at explaining technical concepts in a way that was easier to understand.

What about the process of working through a design document did you find helpful when developing the code?

Working through the design document helped me think about the project before jumping straight into coding. It made me slow down and look at the requirements, how the different parts of the system would work together, and what challenges might come up later. Having a plan made the coding assignments easier because I already had a better understanding of how everything fit together.

If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?

If I could go back and revise one part, I would probably spend more time on the evaluation section. I think I could have gone into more detail about the strengths and weaknesses of each platform. At the beginning of the course I was still learning a lot about operating systems, so I feel like I understand those differences much better now than I did when I first started.

How did you interpret the user's needs and implement them into your software design? Why is it so important to consider the user's needs when designing?

I focused on what The Gaming Room wanted the software to accomplish and used those requirements to guide my design decisions. Since they wanted a game that could support multiple users across different platforms, I looked for solutions that would provide flexibility, scalability, and security. Understanding the user's needs is important because the software is being built for them. If the design does not solve the client's problem, then it does not matter how good the technology is.

How did you approach designing software? What techniques or strategies would you use in the future to analyze and design a similar software application?

I approached the design by breaking the project into smaller sections and focusing on one requirement at a time. The UML diagram, platform comparisons, and design patterns helped me organize my thoughts and understand how the system should work. In the future, I would continue using planning documents, diagrams, and requirement analysis because they help create a clearer picture of the project before development begins. This course showed me that spending time on design upfront can save a lot of confusion later.
